# Guía de Contribución para Match Paths 🚀

¡Gracias por interesarte en mejorar Match Paths! Las contribuciones de la comunidad ayudan a que este proyecto de entrenamiento cognitivo sea más accesible y robusto.

## ¿Cómo puedo colaborar?

### 1. Reportar Errores (Bugs)
Si encuentras un fallo en la lógica del juego o un problema de diseño visual, abre un **Issue** detallando:
* Una descripción clara del problema.
* Los pasos exactos para reproducir el error.
* Capturas de pantalla si es un fallo de la interfaz (UI).

### 2. Proponer Nuevas Características
¿Tienes ideas para nuevos niveles, temáticas de vehículos o mejoras en los efectos de sonido? Abre un **Issue** etiquetado como `enhancement` para discutirlo antes de escribir el código.

### 3. Enviar un Pull Request (PR)
Si ya tienes el código para solucionar un problema o añadir una función:
1. Haz un **Fork** de este repositorio.
2. Crea una rama para tu mejora (`git checkout -b feature/nueva-mejora`).
3. Realiza tus cambios y asegúrate de que el código esté limpio y comentado.
4. Haz un *Push* a tu rama (`git push origin feature/nueva-mejora`).
5. Abre un **Pull Request** hacia nuestra rama `main` explicando detalladamente tus cambios.

## Estilo de Código
* Mantén el código de React modular y reutilizable.
* Utiliza clases de Tailwind CSS semánticas para el diseño responsivo.
* Asegúrate de que las transiciones de CSS no afecten el rendimiento de las animaciones de las cartas.
